## React 核心概念与现代开发方式

> 这篇主要解决一个问题: 现在面试里到底该怎么理解 React。重点不是背 API，而是建立一套比较稳的回答框架。

<!-- more -->

## 一、React 是什么？

一句话概括:

> `React` 是一个用于构建用户界面的 JavaScript 库，它通过组件化、声明式和单向数据流来组织复杂前端应用。

面试里可以继续往下展开三层:

+ `React` 只聚焦视图层，不是完整的大而全框架
+ 它强调组件拆分和状态驱动视图
+ 它依赖生态来补齐路由、状态管理、构建、请求等能力

所以更准确地说，`React` 的强项不是“什么都内置”，而是“围绕 UI 这件事做得很彻底”。

## 二、React 的设计思想有哪些？

这是 React 面试里很适合拉开差距的一题，建议从下面几个关键词回答。

### 1. 声明式

开发者描述“当前状态下界面应该长什么样”，而不是手动命令式操作 DOM。

例如:

+ 状态是 `isLogin = true`，界面显示用户信息
+ 状态变成 `false`，界面显示登录按钮

重点在于:

+ 你关心的是状态和 UI 的映射关系
+ DOM 更新细节交给 React 处理

### 2. 组件化

React 把 UI 拆成一个个可组合的组件，每个组件负责相对独立的一段视图和逻辑。

组件化的价值:

+ 提升复用性
+ 提升可维护性
+ 更容易做职责划分
+ 更适合多人协作

### 3. 单向数据流

父组件通过 `props` 把数据传给子组件，数据流向通常是单向的。

这意味着:

+ 状态来源更清晰
+ 调试链路更明确
+ 组件之间耦合更低

子组件如果想影响父组件，通常通过回调函数、事件意图上报或者状态管理方案来完成。

### 4. 组合优于继承

React 官方一直更推崇“组合”而不是“继承”。

常见体现:

+ 通过 `children` 做内容组合
+ 通过自定义 Hook 复用逻辑
+ 通过高阶组件、Render Props 或组合组件模式复用能力

## 三、React 为什么流行？

可以从工程和心智模型两方面回答。

### 工程层面

+ 生态成熟，社区体量大
+ 可与各种工具链灵活组合
+ 对大型前端应用的组件化开发非常友好
+ React Native 让它在跨端方向也有延展性

### 心智模型层面

+ UI = f(state)
+ 一切围绕组件和状态组织
+ 逻辑可拆成 Hook，结构更容易复用

## 四、React 和传统 jQuery / 命令式开发的区别

这是一个很适合面试中顺手展开的对比题。

传统命令式开发更像:

+ 找到 DOM
+ 修改 DOM
+ 绑定事件
+ 手动维护 DOM 和数据一致性

React 的思路更像:

+ 维护状态
+ 根据状态生成 UI 描述
+ 状态变化后重新计算界面
+ 由 React 负责把变化同步到真实 DOM

核心区别不只是“有没有虚拟 DOM”，而是:

> React 把视图开发从“手动操作节点”切成了“描述状态到视图的映射关系”。

## 五、函数组件为什么成为主流？

早期 React 中类组件很常见，但现在面试和实际项目里主流都是函数组件。

原因主要有这些:

+ 函数组件写法更简洁
+ 更容易复用逻辑，自定义 Hook 非常自然
+ 更符合 React 后续并发、可中断渲染的发展方向
+ 不再依赖 `this`，减少了很多心智负担

面试里不要答成“类组件不能用了”。

更准确的说法是:

+ 类组件依然可运行
+ 但社区和官方推荐主流已经转向函数组件 + Hooks
+ 新能力和最佳实践基本都围绕函数组件展开

## 六、React 项目现在常见的技术栈是什么？

这题通常不是死记硬背，而是体现你是否了解现代 React 开发。

常见组合大致如下:

+ 视图层: `React`
+ 路由: `React Router`
+ 状态管理: `Redux Toolkit`、`Zustand`、`Jotai`，或者服务端状态用 `TanStack Query`
+ 构建工具: `Vite`、`Next.js`
+ 样式方案: `CSS Modules`、`Sass`、`Tailwind CSS`、`styled-components`、`emotion`
+ 类型系统: `TypeScript`
+ 表单方案: `React Hook Form`、`Formik`
+ 测试: `Vitest`、`Jest`、`Testing Library`

如果是面试中回答，可以强调:

> 当前 React 生态里，客户端状态和服务端状态通常会分开看，不会再把所有状态都塞进一个全局 store。

## 七、React 面试里最常见的题型有哪些？

大致可以分成下面几类。

### 1. 基础认知题

+ React 是什么
+ JSX 是什么
+ 组件化怎么理解
+ 受控组件和非受控组件

### 2. Hooks 题

+ `useState`、`useEffect`、`useMemo`、`useCallback`
+ Hook 闭包陷阱
+ 依赖数组怎么理解
+ 自定义 Hook 如何设计

### 3. 性能优化题

+ 为什么组件会重复渲染
+ `React.memo` 有什么用
+ `useMemo` 和 `useCallback` 的区别
+ 列表 `key` 为什么重要

### 4. React 18 题

+ 自动批处理
+ 并发渲染
+ `useTransition`
+ `useDeferredValue`
+ `StrictMode` 为什么开发环境会执行两次

### 5. 状态管理题

+ `props` drilling 怎么解决
+ Context 的优缺点
+ Redux 工作流程
+ 为什么现在更推荐 `Redux Toolkit`

### 6. 源码原理题

+ Fiber 是什么
+ 虚拟 DOM 和 Diff
+ `setState` 为什么异步感
+ 调度、render、commit 各做什么
+ Hooks 为什么不能放在条件语句里

## 八、现代 React 学习顺序建议

如果要建立适合面试和项目的知识体系，建议按这个顺序走:

1. 先理解组件、状态、事件、列表渲染、条件渲染
2. 再掌握 Hooks，尤其是 `useState`、`useEffect`、`useRef`
3. 再补 Context、路由、表单、状态管理
4. 再学习性能优化和 React 18 并发相关能力
5. 最后再看 Fiber、调度器、更新队列、Hooks 实现原理

这样学的好处是:

+ 先能写
+ 再能解释
+ 最后能说原理

## 九、面试回答 React 的通用模板

很多 React 题都可以用下面这个结构回答:

### 1. 先讲定义

一句话说清它是什么。

### 2. 再讲作用

它解决什么问题，适合什么场景。

### 3. 再讲原理

底层大概怎么工作，核心流程是什么。

### 4. 再讲优缺点或注意点

什么时候该用，什么时候别滥用。

例如回答“React.memo 是什么”时就可以这样说:

> `React.memo` 是一个用于函数组件的浅比较优化手段。当组件接收的 `props` 没变化时，它可以跳过本次渲染结果计算，从而减少不必要的重新渲染。它适合纯展示组件或者渲染成本较高、且 `props` 比较稳定的组件，但如果传入的对象和函数每次都新建，优化效果就会大打折扣。

## 十、这一篇的结论

如果面试官问“你怎么理解 React”，一个比较稳的总结可以这样说:

> `React` 本质上是一个面向组件的 UI 库，它通过声明式、单向数据流和组件化来管理复杂界面。现代 React 的主流开发方式已经从类组件转向函数组件加 Hooks，同时在 React 18 中进一步加入并发渲染、自动批处理等能力。实际面试里，除了 API 使用，更重要的是能讲清 React 的状态驱动模型、渲染流程、性能优化思路以及 Fiber 这一套底层机制。
