## script setup 是个啥？
它是 Vue3 中用于编写 Composition API 的单文件组件语法糖。`<script setup>` 里的代码会被编译到组件的 `setup()` 中执行，相比传统写法更简洁。
需要注意的是，暴露给模板使用的不是“ES 模块导出”，而是 `<script setup>` 里的顶层绑定，比如变量、函数、导入的组件等。
使用方式极其简单，仅需要在 script 标签加上 setup 关键字即可。示例：
``` <script setup></script> ```

## 组件自动注册
在 `<script setup>` 中，引入的组件可以直接在模板里使用，无需再通过 `components` 选项注册。组件名通常会根据文件名推导；如果需要显式声明组件名或其他组件选项，Vue 3.3+ 可以使用 `defineOptions`，更早的版本再使用一个普通的 `<script>` 标签补充配置。示例：
```
<template>
    <Child />
</template>

<script setup>
import Child from './Child.vue'
</script>
```
+ Vue 3.3+ 可以这样定义组件选项：
```
<script setup>
defineOptions({
  name: 'ChildContainer'
})
</script>
```



## 组件核心 API 的使用
使用 props
通过 `defineProps` 声明当前组件接收的 `props`，并得到 `props` 对象。它是编译宏，只能在 `<script setup>` 中使用，不需要手动导入。示例：
```
<script setup>
  const props = defineProps({
    title: String,
  })
</script>
```

## 使用 emits
使用 `defineEmits` 声明当前组件会触发的事件，并拿到 `emit` 方法。它同样是编译宏，不需要手动导入。示例：
```
<script setup>
  const emit = defineEmits(['change', 'delete'])
</script>
```

## 获取 slots 和 attrs
早期 RFC 中出现过 `useContext` 这种写法，但正式发布后的 Vue 3 中应使用 `useAttrs` 和 `useSlots`。示例：
```
<script setup>
  import { useAttrs, useSlots } from 'vue'

  const attrs = useAttrs()
  const slots = useSlots()
</script>
```

## defineExpose API
父组件仍然可以通过模板 `ref` 拿到子组件实例，但当子组件使用 `<script setup>` 时，组件内部默认是“关闭”的，父组件拿不到里面的变量和方法。
如果需要对外暴露 `<script setup>` 中的数据和方法，需要使用 `defineExpose`。它是编译宏，不需要手动导入。示例：
```
<script setup>
const a = 1
const b = 2

defineExpose({
  a,
  b
})
</script>
```
属性和方法无需返回，直接使用！
这可能是带来的较大便利之一，在以往的写法中，定义数据和方法，都需要在结尾 return 出去，才能在模板中使用。在 script setup 中，定义的属性和方法无需返回，可以直接使用！示例：
```
<template>
  <div>
   	<p>My name is {{name}}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue';

const name = ref('Sam')
</script>
```

## Vue3 defineProps、defineEmits、defineExpose 的作用
这几个 API 都是 `<script setup>` 中常见的编译宏。


### defineProps - 组件之间传值
```
// 父组件
<template>
  <Child :val="val"></Child>
</template>

// 子组件
<script setup>
const props = defineProps({
  val: {
    type: String,
    default: ""
  }
})
</script>

```


### defineEmits - 子组件向父组件事件传递
```
// 父组件
<template>
 <Child  @success="success"></Child>
</template>
<script setup>
function success (e) {
  console.log(e)
}
</script>

//子组件
<template>
 <button @click="handleClick">点击按钮</button>
</template>
<script setup>
const emits = defineEmits(['success'])
function handleClick () {
  emits("success", "子组件向父组件传递数据")
}
</script>
```


### defineExpose - 子组件暴露自己的属性或方法
```
// 父组件
<template>
  <Child ref="RefChildExpose"></Child>
  <button @click="touchButton">点击使用子组件</button>
</template>
<script setup>
import { ref } from 'vue'

const RefChildExpose = ref(null)
function touchButton () {
  // 使用子组件方法
  RefChildExpose.value.show()
  // 输出子组件属性
  console.log(RefChildExpose.value.count)
}
</script>

// 子组件
<script setup>
function show () {
  console.log('显示')
}
defineExpose({
  show,
  count: 1
})
</script>

```

## Vue3 获取 DOM 的推荐方式

```
<template>
  <div ref="root">This is a root element</div>
</template>

<script setup>
import { onMounted, useTemplateRef } from 'vue'

// Vue 3.5+ 推荐写法
const root = useTemplateRef('root')

onMounted(() => {
  console.log(root.value) // <div>This is a root element</div>
})
</script>


```

```
<script setup lang="ts">
import { onMounted, ref } from 'vue'
import { ElForm } from 'element-plus'

// Vue 3.5 以下版本可以这样写
const formRef = ref<InstanceType<typeof ElForm> | null>(null)

onMounted(() => {
  formRef.value?.validate((valid) => {
    console.log(valid)
  })
})
</script>

<template>
  <el-form ref="formRef"></el-form>
</template>

```
通常更推荐使用模板 `ref` 获取 DOM 或组件实例，不建议把 `getCurrentInstance().proxy` / `$refs` 当成主要方案，尤其是在 TypeScript 场景下可读性和类型提示都比较一般。

## 获取 this
Vue2 中每个组件里使用 this 都指向当前组件实例，this 上还包含了全局挂载的东西、路由、状态管理等啥啥都有
而在 Vue3 的 `setup` / `<script setup>` 中，`this` 默认是 `undefined`。如果确实需要拿到当前组件实例，可以使用 `getCurrentInstance()`，但它更适合少量底层场景，不建议把它当成 `this` 的完全替代品。如下：
```
<script setup>
import { getCurrentInstance } from 'vue'

const instance = getCurrentInstance()

// 当前组件所在应用上的全局属性
const global = instance?.appContext.config.globalProperties
</script>

```
如果只是想使用路由、状态管理或全局注入能力，更推荐优先使用对应的组合式 API，例如 `useRouter()`、`useRoute()`、`inject()` 等。
## setup注意要点

`setup` 的执行时机在 `beforeCreate` 之前，所以在 `setup` 里拿不到组件实例上的 `this`，打印结果会是 `undefined`。
普通写法下，`setup` 可以接收两个参数：第一个是 `props`，第二个是 `context`。`context` 里常用的有 `attrs`、`slots`、`emit`，可以理解为对应 Vue2 里的 `this.$attrs`、`this.$slots`、`this.$emit`。
如果使用的是 `<script setup>`，通常会直接使用 `defineProps`、`defineEmits`、`useAttrs`、`useSlots`，而不是手动接收 `context`。



## ref与reactive的区别

`ref` 常用于定义单个响应式值，既可以是基本类型，也可以是对象类型。在 JS 代码里读写 `ref` 需要通过 `.value`，但在模板中会自动解包，所以不需要写 `.value`。
`reactive` 用来定义对象类型的响应式数据，也可以处理数组、`Map`、`Set` 等。它返回的是代理对象，在 JS 和模板中都不需要 `.value`。
如果把对象传给 `ref`，Vue 内部也会把这个对象转换成响应式对象。



## 对于 Vue3 中的响应式

通过Proxy（代理）: 拦截对象中任意属性的变化, 包括：属性值的读写、属性的添加、属性的删除等。
通过Reflect（反射）: 对源对象的属性进行操作。`Reflect` 是 JavaScript 的内置对象，不是 `window` 独有的能力。
```
const p=new Proxy(data, {
// 读取属性时调用
    get (target, propName) {
    	return Reflect.get(target, propName)
    },
//修改属性或添加属性时调用
    set (target, propName, value) {
    	return Reflect.set(target, propName, value)
    },
//删除属性时调用
    deleteProperty (target, propName) {
    	return Reflect.deleteProperty(target, propName)
    }
}) 
```

```
beforeCreate===> 直接写在 setup 中
created=======> 直接写在 setup 中
beforeMount ===>onBeforeMount
mounted=======>onMounted
beforeUpdate===>onBeforeUpdate

## toRef与toRefs

```
<template>
  <div class="home">
    <h1>当前姓名:{{names.name}}</h1>
    <h1>当前年龄:{{names.age}}</h1>
    <h1>当前薪水:{{names.job.salary}}K</h1>
    <button @click="names.name+='!'">点击加!</button>
    <button @click="names.age++">点击加一</button>
    <button @click="names.job.salary++">点击薪水加一</button>
  </div>
</template>

<script>
import {reactive} from 'vue'
export default {
  name: 'Home',
  setup(){
    let names=reactive({
      name:'佩奇',
      age:23,
      job:{
        salary:10
      }
    })
    return {
      names
    }
  }
}
</script>

//是不是一直都是用到代码name.xx，可能你会说，那我就return的时候不这样写，改成这样

// eg1: 

return {
  name:names.name,
  age:names.age,
  salary:names.job.salary
}

// eg2:

// 但是你要是在页面进行操作时就不是响应式了，为什么呢？那是因为你现在暴露出去的是简简单单的字符串，字符串会有响应式吗？肯定没有呀，但是你要是用到了toRef,那就是把name.xx变为响应式，然后操作它时会自动的去修改name里面的数据

return {
  name:toRef(names,'name'),
  age:toRef(names,'age'),
  salary:toRef(names.job,'salary')
}

// eg3: 

//但是有的人可能会说那我为什么不用ref去改变？可能大家会发现在页面里也是响应式的，但是我告诉你们，根本用到的不是names里面的数据，而是你单独定义出来的数据，所以要是这样写，无论怎么修改都不会修改到names里的数据

return {
  name:ref(names.name),
  age:ref(names.age),
  salary:ref(names.job.salary),
}

toRefs

`toRef` 是把某一个属性单独转成 `ref`，`toRefs` 则是把整个响应式对象的每个一层属性都转成 `ref`，这样在解构返回值时更方便。
需要注意，`...toRefs(names)` 只是把第一层属性转成 `ref`，深层对象本身不会继续被自动拆成多个独立 `ref`。
 <h1>当前姓名:{{name}}</h1>
 <h1>当前薪水:{{job.salary}}K</h1>
return {
    ...toRefs(names)
}


```

## Fragment
Vue3 支持组件模板拥有多个根节点，底层会使用 `Fragment` 来承载它们，所以不再像 Vue2 那样强制要求单一根标签。这样可以减少不必要的包裹层级，让模板结构更自然。


## Teleport
teleport 提供了一种有趣的方法，允许我们控制在 DOM 中哪个父节点下渲染了 HTML，而不必求助于全局状态或将其拆分为两个组件。
其实就是可以不考虑你写在什么位置，你可以定义teleport在任意标签里进行定位等（常见操作为模态框），除了body外，还可以写css选择器（id，class）
```
//id定位
 <teleport to="#app">
            <div class="four">
              <div class="five"></div>
            </div>
 </teleport>
//class定位
 <teleport to=".one">
            <div class="four">
              <div class="five"></div>
            </div>
 </teleport>
//示例
<template>
  <div class="one">
    <h1>第一层</h1>
    <div class="two">
      <h1>第二层</h1>
      <div class="three">
        <h1>第三层</h1>
        <teleport to="body">
            <div class="four">
              <div class="five"></div>
            </div>
        </teleport>
      </div>
    </div>
  </div>
</template>

<script>
export default {
  name:'App',
  setup(){
    return {}
  }
}
</script>

<style lang="less">
.one{
  width: 100%;
  background-color: blue;

## 全局API的转移
```

2.x 全局 API（Vue）	 3.x 实例 API (app)

Vue.config.xxxx	    app.config.xxxx

Vue.config.productionTip	移除

Vue.component	    app.component
Vue.directive	    app.directive
Vue.mixin	        app.mixin
Vue.use	            app.use
Vue.prototype	    app.config.globalProperties
```
+ Vue2 中我们要往全局上挂载东西通常就是如下，然后在所有组件里都可以通过 this.xxx 获取到了
```
Vue.prototype.xxx = xxx

```

而 Vue3 中不能这么写了，需要挂到应用实例的 `app.config.globalProperties` 上。比如在 `main.js` 中做全局注册：
```
// main.js
import { createApp } from 'vue'
import App from './App.vue'
const app = createApp(App)
// 添加全局属性
app.config.globalProperties.name = 'xiang'
```
+ 在其他组件中调用
```
<script setup>
import { getCurrentInstance } from 'vue'
const instance = getCurrentInstance()

const global = instance?.appContext.config.globalProperties
console.log(global.name) // xiang
</script>
```


## router

在 Vue3 中，`setup` / `<script setup>` 里不能再直接使用 `this.$router`、`this.$route`，而是通过 `vue-router` 提供的 `useRouter()` 和 `useRoute()` 来获取。

```
import { onMounted } from 'vue'
import {useRouter,useRoute} from "vue-router";

setup(){
  const router= useRouter()
  const route= useRoute()
  function fn(){
    router.push('/about')
  }
  onMounted(()=>{
    console.log(route.query.code)
  })
  return{
    fn
  }
}
```


## 解除绑定
+ Vue2 中一般清除定时器、监听之类的操作有两种方法：

+ 一是用$once 搭配 hook: BeforeDestroy 使用，这个 Vue3 不支持了
+ 二是用 beforeDestroy / deactivated 这俩钩子，Vue3 中只是把钩子函数重命名了一下
```
<script setup>
import { onBeforeUnmount, onDeactivated } from 'vue'

// 组件卸载前，对应 Vue2 的 beforeDestroy
onBeforeUnmount(() => {
    clearTimeout(timer)
    window.removeEventListener('...')
})

// 退出缓存组件，对应 Vue2 的 deactivated
onDeactivated(() => {
    clearTimeout(timer)
    window.removeEventListener('...')
})
</script>

```
