## Vue3核心概念与现代开发方式

> 这一篇先解决一个总问题: 现在面试里该怎么理解 `Vue3`。重点不是背零散 API，而是建立一套比较稳定的回答主线。

<!-- more -->

## 一、Vue3 是什么？

一句话概括:

> `Vue3` 是一个基于组件化和响应式系统构建用户界面的渐进式前端框架。

这个定义里有三个关键词:

+ `组件化`
+ `响应式`
+ `渐进式`

### 1. 组件化

把页面拆成一个个独立组件，每个组件管理自己的模板、状态和行为。

### 2. 响应式

状态变化后，视图会自动同步更新，开发者不需要手动操作 DOM。

### 3. 渐进式

你可以只把它当视图库用，也可以继续配合路由、状态管理、工程化能力构建完整应用。

## 二、Vue3 相比 Vue2 的核心升级是什么？

这是 Vue3 面试最常见的起手题之一。

可以从这几个角度回答:

+ 响应式系统从 `Object.defineProperty` 升级为 `Proxy`
+ 新增 `Composition API`
+ `script setup` 让单文件组件写法更简洁
+ 编译优化更强，如 `Patch Flag`、静态提升、`Block Tree`
+ 对 TypeScript 的支持更自然
+ 状态管理主流从 `Vuex` 转向 `Pinia`

一句话总结可以说:

> `Vue3` 最大的变化是“响应式重写 + 组合式 API + 编译期优化 + 更现代的工程体验”。

## 三、为什么 Vue3 要引入 Composition API？

因为在大型组件里，`Options API` 常常会遇到几个问题:

+ 同一块业务逻辑被拆散在 `data`、`methods`、`computed`、`watch` 里
+ 逻辑复用更多依赖 `mixin`，来源不清晰
+ 类型推导和逻辑抽取都不够自然

而 `Composition API` 的核心价值在于:

+ 按“功能”组织代码，而不是按“配置项”组织
+ 更适合逻辑复用
+ 更适合 TypeScript

## 四、Vue3 现在的主流开发方式是什么？

如果只说“Vue3 支持哪些写法”还不够，面试里更重要的是知道社区主流。

现在更常见的组合是:

+ `Vue3`
+ `script setup`
+ `TypeScript`
+ `Pinia`
+ `Vue Router`
+ `Vite`

其中组件层最主流的写法基本就是:

> `Composition API + <script setup>`

## 五、`Options API` 和 `Composition API` 怎么区分？

### `Options API`

按选项组织:

+ `data`
+ `methods`
+ `computed`
+ `watch`

优点:

+ 上手简单
+ 结构直观

### `Composition API`

按功能组织:

+ 某个业务相关的状态
+ 对应方法
+ 对应副作用
+ 对应计算属性

优点:

+ 逻辑更集中
+ 更适合复杂组件
+ 更容易提取成可复用 `composable`

## 六、为什么很多人说 Vue3 更适合中大型项目？

核心原因不在于“Vue2 不能做大项目”，而在于 Vue3 让大型项目的几个关键点更顺手:

+ 逻辑组织更清晰
+ 类型推导更自然
+ 状态管理更轻量
+ 编译优化和更新机制更成熟

## 七、Vue3 面试里最常见的题型有哪些？

通常可以分成这几类。

### 1. 基础认知题

+ Vue3 相比 Vue2 的变化
+ `Composition API` 和 `Options API`
+ `script setup` 是什么

### 2. 响应式题

+ `ref` 和 `reactive`
+ `toRef` 和 `toRefs`
+ `Proxy` 为什么优于 `defineProperty`

### 3. 副作用和生命周期题

+ `computed` 和 `watch`
+ `watch` 和 `watchEffect`
+ 生命周期变化

### 4. 组件通信和状态管理题

+ `props / emit`
+ `provide / inject`
+ `Pinia` 和 `Vuex`

### 5. 性能优化题

+ 静态提升
+ `Patch Flag`
+ `Block Tree`
+ `Tree Shaking`

### 6. 源码原理题

+ 响应式依赖收集
+ `track` / `trigger`
+ 编译优化
+ 渲染更新流程

## 八、现代 Vue3 的学习顺序建议

建议按下面顺序建立体系:

1. 先掌握组件、模板、指令、事件、列表渲染
2. 再掌握 `ref`、`reactive`、`computed`、`watch`
3. 再补 `script setup`、组件通信、路由、`Pinia`
4. 再补性能优化和源码原理

这样学的好处是:

+ 先能写业务
+ 再能解释为什么这么写
+ 最后再能回答底层原理

## 九、面试回答 Vue3 的通用模板

很多题都可以按这个结构答:

### 1. 先讲定义

它是什么。

### 2. 再讲作用

它解决什么问题。

### 3. 再讲原理

底层大概怎么工作。

### 4. 最后讲边界

适合什么场景，常见误区是什么。

例如回答“`script setup` 是什么”时，可以这样说:

> `script setup` 是 Vue3 中基于 `setup()` 的编译时语法糖。它让组合式 API 的写法更简洁，不需要手动 `return`，顶层绑定可以直接在模板中使用，同时 `defineProps`、`defineEmits` 等宏的使用也更自然。它适合现代 Vue3 项目，是当前主流单文件组件写法。

## 十、一段适合面试现场的总结话术

> `Vue3` 相比 `Vue2`，最大的变化不是单个 API，而是整套开发模型更现代了。它一方面用 `Proxy` 重写了响应式系统，另一方面用 `Composition API` 和 `script setup` 提升了复杂组件的逻辑组织能力，同时在编译阶段加入了静态提升、`Patch Flag`、`Block Tree` 等优化，让更新更精准。再加上 `Pinia`、`Vite` 和更好的 TypeScript 支持，所以 Vue3 更像是一套适合当前工程实践的完整前端开发方案。
