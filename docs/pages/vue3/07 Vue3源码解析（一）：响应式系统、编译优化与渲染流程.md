## Vue3源码解析（一）：响应式系统、编译优化与渲染流程

> Vue3 源码题不一定要求你把实现细节背到函数级别，但至少要能把“响应式怎么收集依赖、状态变化后怎么触发更新、编译优化为什么能让运行时更快”讲顺。

<!-- more -->

## 一、Vue3 源码题最常考什么？

通常集中在三条主线:

+ 响应式系统
+ 编译优化
+ 渲染更新流程

如果能把这三条线串起来，大多数 Vue3 原理题都不会显得太空。

## 二、Vue3 的响应式系统核心是什么？

可以概括为:

+ `Proxy`
+ `Reflect`
+ 依赖收集
+ 派发更新

更简单一点说:

> 读取时收集，修改时触发。

## 三、为什么 Vue3 选择 `Proxy + Reflect`？

### `Proxy`

负责拦截对象操作，例如:

+ `get`
+ `set`
+ `deleteProperty`
+ `has`
+ `ownKeys`

### `Reflect`

负责用更标准的方式完成底层对象操作，让代理逻辑更可控、更接近语言原生语义。

## 四、依赖收集是怎么发生的？

当副作用函数执行时，如果它内部读取了某个响应式属性，Vue 就会把“这个副作用依赖了这个属性”记录下来。

这一过程通常可以抽象为:

+ `track(target, key)`

本质就是建立映射关系:

+ 哪个对象
+ 哪个属性
+ 被哪些副作用使用

## 五、更新触发是怎么发生的？

当响应式属性被修改后，Vue 会找到依赖它的副作用集合，再逐个触发。

这一过程通常可以抽象为:

+ `trigger(target, key)`

所以面试里可以把 Vue3 响应式简化成一句话:

> `track` 记录谁依赖了谁，`trigger` 负责把相关更新重新调度起来。

## 六、`effect` 可以怎么理解？

可以把它理解成一个“可被响应式系统追踪的副作用函数”。

它可能对应:

+ 组件渲染函数
+ `computed`
+ `watchEffect`
+ 其他响应式副作用

## 七、`computed` 在源码层面为什么能缓存？

因为它本质上是一个带惰性执行和脏标记机制的特殊副作用。

依赖不变时:

+ 直接返回缓存值

依赖变化时:

+ 标记为脏
+ 下次访问再重新计算

## 八、`watch` 和 `watchEffect` 在源码层面怎么区分？

### `watch`

+ 明确有监听源
+ 回调何时执行更可控
+ 更像“监听器”

### `watchEffect`

+ 自动收集回调中的依赖
+ 更像“立即执行并自动订阅的 effect”

它们都依赖响应式系统，但封装层次不同。

## 九、编译优化为什么是 Vue3 的关键优势？

因为 Vue3 不只在运行时做事，还在编译阶段就提前分析了模板。

编译器能知道:

+ 哪些节点是静态的
+ 哪些属性是动态的
+ 哪些子节点是真正需要关注的

这样运行时就不必“每次都从头猜”。

## 十、静态提升、`Patch Flag`、`Block Tree` 各自解决什么问题？

### 静态提升

解决“静态节点重复创建”的问题。

### `Patch Flag`

解决“动态节点更新不够精准”的问题。

### `Block Tree`

解决“无意义遍历过多”的问题。

可以把它们理解成一套连续优化:

+ 先在编译阶段标记
+ 再让运行时更聚焦地更新

## 十一、Vue3 的渲染流程怎么理解？

可以按这条链路回答:

1. 模板被编译成 render 函数
2. render 函数执行生成虚拟节点
3. 响应式依赖在渲染过程中被收集
4. 数据变化后触发副作用重新执行
5. 新旧虚拟节点比较后，更新真实 DOM

如果还想再说得进阶一点，可以补充:

> Vue3 不是单纯“每次全量重渲染”，而是依赖响应式追踪和编译标记，让更新范围更精准。

## 十二、为什么说 Vue3 是“响应式驱动 + 编译优化”双轮模型？

因为它的性能和体验不是只靠其中一个点。

+ 响应式系统决定“谁变了、谁该更新”
+ 编译优化决定“更新时怎么少做无效工作”

这也是它和很多纯运行时框架相比的一个明显特点。

## 十三、Vue3 源码题高频追问

### 1. `Proxy` 是不是比 `defineProperty` 绝对更快？

不要答绝对。更准确地说是:

+ 代理能力更强
+ 惰性处理更自然
+ 整体实现更适合现代响应式框架

### 2. `computed` 和 `watch` 都是响应式副作用吗？

可以说都依赖响应式副作用机制，但语义不同:

+ `computed` 侧重派生值
+ `watch` 侧重副作用监听

### 3. Vue3 更新为什么更精准？

因为有响应式依赖收集，再加上编译期标记，更新路径比过去更细粒度。

## 十四、一段适合面试现场的总结话术

> Vue3 底层可以概括为两部分：一部分是响应式系统，基于 `Proxy`、`Reflect`、`track` 和 `trigger`，实现读取时收集依赖、修改时触发更新；另一部分是编译优化，编译器会提前分析模板，生成 `Patch Flag`、静态提升和 `Block Tree` 等信息，让运行时更新更精准。最终组件渲染时，render 函数既承担 UI 描述生成，也承担依赖收集，所以 Vue3 的更新性能并不是只靠 Diff，而是响应式系统和编译器一起工作的结果。
